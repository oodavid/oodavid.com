Todays Blog post has been brought to you in the format of an exremely busy person.

Brevity is the new levity.

  • Create a simple Google Spreadsheet
  • Publish it (it can still be privately editable)
  • Download my script (github gist)
  • Tweak the settings in the code
    • Link to the published CSV file
    • Set the column names
    • Run the page
  • Et viola! You have a formatted JSON object for use in your projects.

Making use of the tool

The usefulness might not be immediately apparent...

When working on NexGen I am required to complete projects and add features in a rapid manner. We have all-but-removed the need for meetings instead agreeing that prototype development by gut-instinct eclipses the time spent deliberating and group planning. Standard Agile stuff really.

The Setup

  • Our Mobile App has many (Risk) definitions in JSON format.
  • These Risks are created and managed by one of our Directors on Google Drive.
  • Changes are infrequent, but important.

The Solution

  • The Director continues to work in his usual manner
  • I use "onchange" notifications to make sure I know when he updates the spreadsheet.
  • This script is then manually used by me, and the mobile app updated.
